### 2. Приведем описание параметров Apptio TBM Unified Model
| Хэдеры | Описание |
| --- | --- |
| `IT Tower` | Это группа, ответственная за управление инфраструктурой сервиса, его поддержку и характеристику. Примеры IT Tower - Storage, Machine Learning, Database, Security, Analytics, Compute.|
| `Service Family` | Это группа сервисов, которая выполняет конкретную инфраструктурную задачу и связана с конкретным приложением. Примеры Service Family - Application services, Management or provisioning, GPU-powered acceleration, In-memory caching service и другие...|
| `Service Type` | Это тип сервиса внутри определенной группы сервисов. Примеры Service Type -  Elastic File System, Elastic Inference, ElastiCache, GuardDuty, Kinesis и другие... |
| `Service Sub Type` | Это подтип сервиса с определенной функцией. Примеры Service Sub Type -- Fees, Acceleration, Cache Nodes, Security & Compliance и другие... |
| `Service Usage Type` | Это тип использования сервиса с учетом подтипа, также тип запросов, которые может выполнять пользователь с использованием конкретного сервиса. Примеры Service Usage Type - Standard Timed storage, Infrequent Access storage и другие... |

### 3. Рассмотрим приведенные в таблице сервисы
| Хэдеры | Описание |
| --- | --- |
| `Azure App Center` | Платформа для разработки, тестирования и развертывания мобильных приложений с возможностями непрерывной интеграции, тестирования и обеспечения.|
| `Azure App Configuration` | Управление конфигурацией приложения в централизованном хранилище, позволяющее динамически изменять параметры приложения без его пересборки.|
| `Azure App Service` | Управляемая платформа для развертывания веб-приложений, мобильных бэкэндов и API. Предоставляет высокоуровневые службы для упрощения разработки.|
| `Azure Application Insights` | Сервис мониторинга и отслеживания производительности приложений, предоставляющий данные о запросах, ошибках и использовании ресурсов.|
| `Azure Automation` | Позволяет автоматизировать управление и конфигурацию ресурсов в облаке Azure и локально.|
| `Azure Bastion` | Обеспечивает безопасный удаленный доступ к виртуальным машинам через браузер, устраняя необходимость открытия общедоступных портов.|
| `Azure Firewall` | Управляемый брандмауэр для контроля и фильтрации сетевого трафика в облаке Azure.|
| `Azure Notification Hubs` | Сервис для отправки миллионов уведомлений на мобильные устройства (iOS, Android, Windows) или другие платформы.|
| `Azure Power BI` | Сервис для визуализации и анализа данных, позволяющий создавать дашборды и отчеты.|
| `Azure Site Recovery` | Предоставляет возможности резервного копирования и восстановления для приложений и рабочих нагрузок.|
| `Azure Virtual Machines` | Позволяет развертывать виртуальные машины с различными операционными системами в облаке Azure.|
| `Azure SQL Data Warehouse (Synapse Analytics)` | Облачный сервис для аналитической обработки данных с масштабированием хранилища данных.|

### 5. Сравним с российским провайдером
| Хэдеры | Описание |
| --- | --- |
| `Azure App Center` | `Yandex Cloud Apps`|
| `Azure App Configuration`  | `Yandex Cloud Apps`|
| `Azure App Service` | `Yandex Cloud Apps`|
| `Azure Application Insights` | `Yandex Monitoring`|
| `Azure Automation`  | `Yandex Resource Manager`, но не весь функционал Azure реализован |
| `Azure Bastion`  | `Yandex Compute Cloud`, внутри сервиса есть этот функционал, отдельно сервиса нет |
| `Azure Firewall` | `Yandex Compute Cloud`, внутри сервиса есть этот функционал, отдельно сервиса нет |
| `Azure Notification Hubs` | такого нет, но есть реализация расслыки по почте |
| `Azure Power BI` | `Yandex DataLens`|
| `Azure Site Recovery`  | `Yandex Cloud Backup` предназначен для виртуалок |
| `Azure Virtual Machines` | `Yandex Compute Cloud`|
| `Azure SQL Data Warehouse (Synapse Analytics)` | `Yandex DataSphere`|

### Вывод: многие сервисы Azure можно заменить российским аналогом Yandex Cloud, но многие сервисы Яндекса объединяют в себе функции, выполняемые отдельными сервисами Azure. Эффективнее ли это - вопрос. Полностью "переехать" вряд ли получится, так как Azure предлагает огромный функционал, который Yandex Cloud только выдвигает в Previw.
